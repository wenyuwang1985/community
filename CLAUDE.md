# AI 开发规范

本项目使用 AI 辅助开发。所有 AI 工具（Claude Code、Codex、Kimi Code 等）在参与开发时，必须遵守以下规则。

## 项目概述

社区生活系统 — 街镇级社区连接平台。前后端分离架构：
- `server/`：Go + Gin 后端
- `client/`：uni-app + Vue 3 前端
- 数据库：PostgreSQL + PostGIS
- 缓存：Redis
- 主键策略：Snowflake ID（应用层生成）

## 核心规则

### 1. 文档先行

开发任何功能前，必须先在对应的 docs 目录写设计文档，确认后再写代码：
- 后端功能 → `server/docs/`
- 前端功能 → `client/docs/`
- 数据库变更 → `server/docs/sql/`

设计文档应包含：目标、方案、涉及的文件、接口定义（如适用）。

### 2. 变更说明

每次改动完成后，说明：
- 改了哪些文件
- 为什么改（解决什么问题 / 实现什么功能）
- 有无需要注意的副作用

### 3. 最小实现

- 严格按需求实现，不自作主张加功能、加抽象、加防御性代码
- 一个任务做一件事，不顺手重构不相关的代码

### 4. 语言规范

- 代码注释：中文
- Commit message：中文
- 文档：中文
- 变量/函数命名：英文（Go 标准风格）

## 技术约定

- Go 代码遵循 project-layout 目录结构
- API 路由统一前缀 `/api/v1/`
- 错误处理使用统一响应格式
- SQL 文件按编号命名：`001_xxx.sql`、`002_xxx.sql`
- 本地开发环境通过根目录 `docker-compose.yml` 启动
- 服务端编译产物在 `server/bin` 目录
